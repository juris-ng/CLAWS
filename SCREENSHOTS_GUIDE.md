# App Screenshots Guide

## Required Screenshots

### iOS (iPhone)
- Size: 1290×2796px (iPhone 15 Pro Max) or 1242×2688px (iPhone 13 Pro Max)
- Minimum: 3 screenshots
- Maximum: 10 screenshots
- Format: PNG or JPEG

### iOS (iPad)
- Size: 2048×2732px (12.9" iPad Pro)
- Minimum: 3 screenshots
- Maximum: 10 screenshots

### Android
- Size: 1080×1920px minimum
- Minimum: 2 screenshots
- Maximum: 8 screenshots
- Format: PNG or JPEG

## Screenshot Order (Recommended)

1. **Home Screen** - Show main feed with petitions
2. **Petition Detail** - Show voting and commenting
3. **Create Petition** - Show creation flow
4. **Leaderboard** - Show gamification
5. **Profile** - Show user profile and points
6. **Notifications** - Show real-time updates
7. **Admin Dashboard** (optional) - If highlighting admin features

## Screenshot Design Tips

### Content
- Show real-looking data (not "Lorem Ipsum")
- Use diverse user names and avatars
- Show realistic vote counts and engagement
- Display notifications badge
- Show points and levels

### Annotations (Optional)
- Add text overlays explaining features
- Use your brand colors (#0066FF)
- Keep text minimal and readable
- Use consistent font (SF Pro for iOS, Roboto for Android)

### Tools
- Use iOS Simulator / Android Emulator
- Screenshot with Cmd+Shift+4 (Mac) or Windows+Shift+S (Windows)
- Edit with Figma, Canva, or Screenshot Maker apps
- Use https://www.screely.com/ for beautiful mockups

## Screenshot Template Structure
[Top 20% - White space with app name/tagline]
"Make Your Voice Heard"

[Middle 60% - Actual app screenshot]
[Clean, centered phone frame]

[Bottom 20% - Feature highlight text]
"Vote on petitions that matter to you"

## Localization
If supporting multiple languages:
- Provide screenshots in each language
- Translate overlay text
- Keep UI elements localized
